# 0.5.1

A patch release on the `0.5.x` line — additive primitives, one element fix, and a pre-release
cleanup. Per the [versioning policy](../../RELEASING.md), additive features land in a patch pre-1.0;
the public API surface stays frozen (`check:api` green).

> Draft release notes — paste as the GitHub release body when `v0.5.1` is cut. This file does not
> itself bump versions or tag.

## Highlights

### Field channels — `addField` / `sampleField`

The field can now read data you already own. `addField(name, sampler)` registers an external scalar
field — terrain height, soil moisture, a heat map — as a named **channel** the engine samples on its
own read path, read back with `sampleField(name, x, y)`:

```js
const moisture = field.addField('moisture', (x, y) => soil.wetnessAt(x, y));
const wet = field.sampleField('moisture', px, py);   // 0 for an unregistered name
```

It's the open **input** mirror of the render surfaces (`setRender`/`setOverlay` draw *output* layers;
a channel is a field the engine *reads*). Pull-based and never cached, so the data stays yours and
always current. Returns a `FieldChannelHandle` to swap the sampler live or remove the channel.
Mirrored through vanilla / `<field-root>` / three. New guide: **Field channels**
(`/docs/field-channels`).

### Reactive `BodyHandle.set`

Programmatic bodies (`addBody`) gained the live setter the Three.js `FieldBody` already had:

```js
const lure = field.addBody({ tokens: 'attract', strength: 0.8, rect });
lure.set({ strength: 0.3 });   // live — no rescan, no remove + re-add
```

`set({ strength, range, angle, spin, color })` mutates force params on the measure cadence; `color`
re-tints the carried pigment. A structural change (different `tokens`) still needs remove + `addBody`.

## Fixed

- **`<field-root>` now forwards `FieldOptions.depth`.** `<field-root depth="40">` was a no-op —
  `depth` had no attribute, getter, or forwarding, though core/react/vanilla/three all supported it.
  The element now opens the z-volume; its option object is table-driven so an option can't be silently
  dropped again (a drift-guard test pins `observedAttributes` to that table).

## Also in this release (previously unreleased)

- `FieldOptions.dprCap` + `setDprCap(cap)` — a configurable render-resolution ceiling, the dominant
  fill-rate lever on retina.
- `FieldHandle.addBody(spec)` — first-class programmatic bodies for non-DOM hosts.

## Removed

- `setSurfaces` / `getSurfaces` / `SurfacePlan` and `readParticleColors` were pulled before they
  shipped. A design review found both were the rigidity they were meant to solve — `setSurfaces` is a
  fixed-key struct where an open `registerOverlay` registry belongs (the output mirror of `addField`),
  and `readParticleColors` was a half-feature with no consumer. Both were unreleased, so this is not a
  breaking change. `setRender`/`setOverlay`/`setHeatmap` and `readParticles`/`readParticleIds` are
  unchanged. The replacements are tracked on the board.

## Packaging

This release **restores `@fundamental-engine/elements`** (and the umbrella packages) to npm — they
were absent after the reverted `0.6.0`. All eight packages publish together with provenance.
