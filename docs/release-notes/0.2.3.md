# 0.2.3

> Shipped 2026-06-10.

The invisible-fields cycle. `forces-ui` became **field-ui** — a field-first reframing (rename + aliases,
no physics rewrite) — and on that base the cycle built out the field-first model (contracts, agents,
visual language, recipes, inspection), stood up the twelve-page **invisible-fields example family** whose
render surface is the page's own type, and shipped the engine/platform capabilities that family proved
out: `render: 'none'`, the `QualityGovernor`, `scrollV`, and a renderer-agnostic core reached through an
injected `FieldHost`.

> bump versions or tag.

## Added

- **Renamed to `field-ui`** — `forces-ui` → `field-ui`, `@forces-ui/*` → `@field-ui/*`, with thin
  re-export alias packages, dual-dispatched `field:*`/`forces:*` events, and
  `--field-density`/`<field-root>`/`<field-field>` written alongside their `forces-*` twins. A rename +
  alias pass, not a rewrite: no force formulas, integrator, magnetism, fieldflow, render/heatmap math,
  tokens, or `data-*` authoring changed.
- **`render: 'none'` — the signals-only engine mode** (#297): a field created with `'none'` never
  acquires a canvas context, sizes a backing store, or allocates render scratch; it exists purely as
  signals (`--d`, `--load`, `--lit`, events, `scrollV`). `setRender` out of `'none'` acquires the context
  lazily.
- **The invisible-fields example family** at `/evidence/<slug>` — twelve pages over committed real-data
  snapshots (refreshed weekly by CI) with live in-browser upgrades, provenance chips, and per-page
  signature mechanics; pinned by a Playwright matrix across chromium · webkit · Pixel-7 touch.
- **`QualityGovernor` + the `field:quality-tier` event** — adaptive frame-budget tier detection (0–3,
  asymmetric escalation/recovery); the `<field-root>` runtime feeds it, skips discontinuity frames,
  resets on `visibilitychange`, and throttles its own tick at tiers 2–3.
- **`FieldHandle.scrollV()` + `--field-scroll-v`** — the engine's eased page-scroll velocity, mirrored to
  `:root` by the platform write phase. Also `FieldHandle.setVisible(on)` (skip draw while sim/feedback
  stay live), `FieldHandle.particleCount()` / `.energy()` diagnostic accessors,
  `FeedbackRegistry.cssWritesLastFrame()`, and `PlatformRuntime.attachHandle()`.
- **`@field-ui/platform` — the DOM-binding layer** (PR-A…D7): measurement / state / feedback /
  relationship / visual-binding / overlay registries, the explicit six-phase `FrameScheduler`, platform
  lint, `bindData()`, plus `withFlip()` (FLIP reflow helper) and `allocateAttention()` (conserved
  water-filling allocation, unit-tested for exactness). `<field-root>` runs on the platform runtime by
  default.
- **Field-first model build-out** (migration-plan Phases 4–8, engine-side, pure, node-tested): formal
  `core/contracts` with a validated `ForcePassport` for every force; the `core/agents` FieldAgent model
  (element / relationship / user / layout / data + a hysteretic EventAgent); `core/visual`
  metric→appearance mappings + lint + semantic-text fallback; `core/recipes` (SceneRecipe schema, intent
  compiler, gallery, Explain / Diff) expanded to a 64-recipe catalog; and `core/inspect` (deterministic
  snapshot regression, budget inspector, system report).
- **Natural Field Translation System** — the four-field classification model (Gravity→importance,
  EM→polarity, Strong→binding, Weak→transformation) + canonical doc, with natural-field badges and Lab
  filtering.
- **Diagnostics + surfaces** — on-canvas topology / inspector / causality / prediction render modes; the
  `core/diagnostics` module (energy, potential, probes, heatmap variants); PNG/SVG field export; and
  **Field Surfaces** — `setOverlay()` renders the visualization in front of content (over + under =
  immersive).
- **Controlled flow-field API** (`flowTo` / `clearFlow`), the in-repo Vite **starter** app, the
  `@field-ui/vanilla` framework-free wrapper, and a generated **Custom Elements Manifest**.
- **Frozen 0.x public API surface**, enforced in CI (`check:api`), plus `publint`/`attw` packaging
  checks, a `check:dist` import smoke check, and a CI provenance publish workflow.

## Changed

- **`@field-ui/core` imports zero DOM** — the engine is renderer-agnostic, reached through an injected
  `FieldHost`; legacy DOM glue is quarantined behind a boundary guard.
- **Six canonical force tokens renamed to functional terms** — `vortex→swirl`, `spring→tether`,
  `emitter→jet`, `drag→viscosity`, `reflect→wall`, `absorb→sink` (a hard rename of `data-body` values;
  `attract`/`repel`/`stream` unchanged). The capture-radius attr stays `data-absorb`, the accretion var
  stays `--load`.
- **`--load` is the canonical accretion var** (aliasing `--mass`); `threads`' depth var renamed `--depth`
  to stop colliding with the engine's `--d` channel.
- The site was re-centered on the **data-attribute authoring model** (teach the contract, then the
  physics), with the natural fields taught before designed forces; `/eli5` added as a plain-language
  second homepage; the 8-paper research family published at `/research` (KaTeX math).

## Fixed

- **Every force can disturb the resting field** — the tear pass had a hardcoded seven-token allowlist, so
  natural primitives and extended forces (`gravity`, `charge`, `thermal`, `diffuse`, …) let the wave
  shimmer ride straight through. Tearing now keys on a force *property*, not a token list.
- **`charge` and `magnetism` act on the live field** — a charge/magnetism body now polarizes nearby
  neutral matter (induction as a field-level pass kept outside the conformance integrator, so the
  "ignores neutral matter" golden stays true).
- **First-class mass no longer corrupts velocity-replacing forces** — mass is applied per-force (additive
  forces scale by `1/m`; `wall`/`jet`/`lens`/`gate` are flagged `kinematic` and set velocity outright),
  so a `wall` bounce no longer drives matter through the wall.
- **Global velocity cap + safety sweep** — the integrator clamps every free particle to the unit-system
  `c`; a conformance safety sweep asserts finite trajectories, bounded heat, and stable particle count.
- **`<field-root>` reacts to live attribute changes and tears down fully** — `observedAttributes` apply
  immediately, and `destroy()`/`disconnectedCallback` release `[data-hot]` engagement listeners so repeat
  mount/destroy doesn't leak handlers.
- Platform honesty fixes: stop hardcoding `relResolved = relTotal`, stop fabricating confidence from
  relationship presence, classify `memory` as a semantic metric (not a physical force), and never default
  `risk` to 0.
