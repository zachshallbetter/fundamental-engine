# 0.4.0

> Shipped 2026-06-13.

The rebrand release: **field-ui → Fundamental**, done as one clean cutover. Every package moves to the
`@fundamental-engine/*` scope, the last `forces-ui` alias window is purged, and the Three.js port gains
`FieldAgent` — a scene object that rides the field.

> bump versions or tag.

## Renamed — `@field-ui/*` → `@fundamental-engine/*`

The project is now **Fundamental** (`fundamental-engine.com`). All packages move to the
`@fundamental-engine` scope; the one-install umbrella becomes the bare package **`fundamental-engine`**
(`npm i fundamental-engine`), replacing `@field-ui/kit` / `@field-ui/field-ui`.

The **primitive is unchanged**: `<field-root>`, `FieldHandle`, `createField`, and the `--field-*` CSS
variables all stay — *fundamental forces act across a field*. Old `@field-ui/*` packages will be
deprecated on npm pointing here.

### Clean break — the `forces-ui` alias window is gone

A hard rename with no compat shims (there are no external consumers yet):

- **Registration protocol** collapsed to `field:*` only (`field:register-body`, …); the `forces:*` twins
  and `FIELD_*` duplicates and their dual dispatch/listen across `shadow.ts` / `field.ts` /
  `platform-runtime.ts` are removed.
- **`--forces-*` CSS vars** removed (density, heatmap-density, `--forces-pull-*` / `--forces-attention`),
  including the platform `FeedbackRegistry` mirror.
- **`forces:*` events** removed (event-agent, platform mirror, DataConsole).
- **`<forces-field>` / `<forces-cell>` / `ForcesField` / `ForcesCell` / `ForcesController` /
  `useForcesField`** removed — `forces-cell.ts` becomes `field-cell.ts`; it *is* the `FieldCell`
  primitive, only the alias was dropped.
- Freeze gate: the alias-window entry is dropped; the 14 remaining frozen entries stay intact.

## Added

### `FieldAgent` — an `Object3D` that rides the field (`@fundamental-engine/three`)

The creatures primitive: a specific scene object (a bee, a fish, a drone) that samples the live field at
its own position each frame, steers along the force (acceleration toward it, drag, a top speed, optional
wander and hover-bob), and writes its world position through the shared `FieldProjection`.

```js
const bee = new FieldAgent(object3d, { projection, sampler, maxSpeed, accel, drag, wander, hover });
bee.update(dt); // per frame
```

Agents are **consumers, not bodies** — they feel the field but exert nothing back unless also registered
with `addBody`. The `sampler` is the `FieldSampler` interface, so an agent can follow a layer, a raw
handle, or any custom blend. Soft edge-bounce keeps agents in-bounds; an injectable `rng` keeps runs
reproducible. Renderer-free and unit-tested. (#426)

## Changed

- **`@fundamental-engine/three`'s `three` peer range relaxed to `>=0.147.0`** (was `>=0.150.0`). An API
  audit shows the package touches only long-stable three symbols — the newest are `InstancedMesh` (r109)
  and `Object3D.clear()` (r123) — and r147 is verified live in a real integration. The declared range now
  matches reality.
