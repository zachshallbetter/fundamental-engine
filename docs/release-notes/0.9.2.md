# 0.9.2

> Shipped 2026-07-01 — the current release.

The substrate release. This is the roadmap batch that turns Fundamental from a runtime you *watch* into
one you can *read and reason about*: a Field Query API, snapshots + diff + causal replay, a
dimension-aware impulse accumulator with angular/temporal/semantic/thermal lanes, dynamic bodies that the
field moves back (with mass and Newtonian recoil), a Projection Registry that reveals state without ever
moving matter, and governance lints that keep the naming lanes honest. The perf story gets its missing
half — an on-hardware fill-rate benchmark beside the pure-Node algorithmic suite — and the README is
reframed around the host-agnostic field runtime the engine has become. Per the
[versioning policy](../../RELEASING.md), additive features land in a patch pre-1.0; everything new here is
**experimental** (opt-in, behavior-preserving, not in the frozen surface).

## Added

### The Field Query & Snapshot substrate (`@fundamental-engine/core`)

- **Field Query API — `field.query(q?)` (critical-path 02).** A read-only, render-agnostic way to ask the
  live field a structured question — a point (`{x, y}` + `radius`), a `DOMRect`-shaped rect, or the whole
  field — and get plain serializable data back: `bodies` (id, rect, tokens, metrics, dimensions, active
  formation), `metrics`, `relationships` (the edge graph by id), and `influences` (per-force Δv at the
  point, from the impulse accumulator). Works headless; never mutates. On every surface (`<field-root>`,
  `<FieldField>`/`useFieldField`, vanilla, three).
- **Query lenses (query phase 2, roadmap 8/10).** A `FieldLens` is a user-supplied declarative scope over
  a query reading — not a preset catalog. Each clause is an allow-list; `field.query({ lens })` scopes the
  live answer and tags it with `lens.id`; the standalone `applyLens(result, lens)` is exported and
  **pure**.
- **Field Snapshot + Diff (MVP) — `field.snapshot(opts?)` / `field.diff(a, b)` (critical-path 03).** A
  portable, versioned, serializable `FieldSnapshot` of *what the field is doing* at a frame — vs a
  screenshot of what it looked like. `diff` reports body/relationship/metric/formation changes; the pure
  `diffFieldSnapshots` is exported. Read-only, headless, privacy-preserving by default (`includeData`
  opt-in).
- **Causal Replay — `field.replay(a, b, opts?)` (critical-path 03 phase 2).** Explains *how* the field
  changed between two snapshots: an ordered, narrated sequence of causes, each with its before/after
  `contribution`. `snapshot({ includeInfluences })` additionally captures per-body force attribution, so
  replay derives **`cause: 'force'`** steps too.

### The dimension-aware accumulator (doc 04 §Step 6)

- **Impulse accumulator core.** The opt-in `Env.accum` (`FieldImpulseAccumulator`) lets a diagnostic or
  query probe read each force's per-particle contribution, captured centrally by the integrator without
  changing the math. `applyAndRecord` is the single capture path; `accumulateAt` gives net + per-force
  attribution at a point; causality/force-vectors/prediction all read it. Byte-identical when absent.
- **Angular / thermal / temporal / semantic lanes.** Per-force Δspin (`acc.angular.z`, with particle
  `orient`/`spin`), thermal contribution, Δage for mortal matter (`acc.temporal.decay`), and the
  conserved-attention multiplier (`acc.semantic.attention`) — all capture-only and behavior-preserving.

### Bodies the field moves back (doc 04 §Steps 4–5, momentum)

- **Body-authority modes.** `data-authority` / `BodySpec.authority`: `anchored` (default), `kinematic`
  (engine writes the visual transform), or `dynamic` (engine owns position/velocity). Reported in
  `query()`/`snapshot()`.
- **Dynamic bodies + recoil.** A `dynamic` body integrates under the net field the others create at its
  centre and writes the result back — bodies bend the field; the field bends them back.
- **First-class body inertial mass (#872).** `mass` gives dynamic bodies an inertial mass ∝ rendered area;
  recoil is `a = F/inertia`. Fixes a latent conflation where recoil scaled by force strength. `mass` off ⇒
  byte-identical.
- **Newtonian own-emission reaction (#873).** Opt-in `reaction`: a dynamic body feels the equal-and-
  opposite of the net impulse it imparts to nearby matter — a directional emitter kicks back like a
  rocket. Default false ⇒ byte-identical.

### Projections & governance (doc 05, critical-path 05)

- **Projection Registry — `field.projections`.** Named projections mapping field *state* to an output
  surface (CSS, dom-attribute, annotation, agent-json, reduced-motion, sound, haptic, …), each declaring
  `channels`, `surfaces`, and accessibility equivalents. **A projection reveals state, it never changes
  it.** Write-phase auto-apply via `projections.bind(...)`; `agentJsonProjection(...)` gives an
  agent/tooling surface whose output is a serializable reading.
- **Governance lints.** `lintProjections()` (reduced-motion / accessibility equivalents), `lintWordLanes()`
  + `field/no-word-in-two-lanes` (`LANE_WORDS` over 36 tokens · 5 formations · conditions · 16 viz modes),
  and `lintDimensionCoupling()` + `field/no-dimension-coupling-without-passport` (`ForcePassport.
  couplesDimensions`; `wall`/`magnetism`/`warp` declare `['linear']`).

### Opt-in integrator & site demos

- **Fixed-timestep integrator (doc 04 §Step 3).** `integrator: 'fixed'` (default `'legacy'`) makes
  per-step decays frame-rate independent (`FRICTION^dt`); byte-identical at the reference rate (`dt === 1`).
- **`/demo` page.** Dynamic-body recoil + a live agent-json projection readout (roadmap 10).

## Changed

- **`dprCap` default stays `2`.** An experimental drop to `1.5` was reverted after on-hardware
  benchmarking (`/perf-bench`) showed no measurable fps benefit — the QualityGovernor already lowers DPR
  adaptively under real load.
- **Aggressive cross-surface substrate coverage (+47 tests).** New `substrate-api.test.ts` in each
  authoring surface verifies `query`/`snapshot`/`diff`/`replay`/`projections` are reachable, shaped, and
  delegate per surface. No behavior change.

## Fixed

- **`@fundamental-engine/dom`: recipe pipeline no longer clobbers `--field-density`.** `apply-recipe.ts`
  excludes the `density` CSS variable from the recipe `varMap`, so the engine's live density signal is
  preserved across recipe changes.

## Performance

- **On-hardware fill-rate benchmark — `/perf-bench` (site).** A real `<field-root>` + mix-blend overlay
  compositing sweep measured on the GPU, confirming the field is fill-rate-bound (not particle-bound) and
  validating the `dprCap` revert.
- **Algorithmic performance suite — `packages/core/bench/` (core).** A deterministic pure-Node benchmark
  of the engine's *algorithmic* cost (full-frame scaling vs particle count, accumulator overhead,
  `query()`/`snapshot()` read cost). Tooling only — not in `dist`.
