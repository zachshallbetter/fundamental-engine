# 0.1.0

> Shipped 2026-06-03.

The first tagged release — the complete engine, feature-complete against the original ROADMAP. This is
`forces-ui`: a framework-agnostic, reciprocal DOM-physics field where every meaningful thing on a page —
a word, a card, a link — becomes a *body* in a particle field rendered on a canvas behind the content.
Bodies bend the field; the field writes its density back into the elements (weight, glow, motion). It
began as the homepage of zachshallbetter.com and outgrew it. An honest initial cut: the spec is stable,
the catalog is complete, and the surfaces are in place — but this is where the story starts, not where it
settles.

## Added

### The engine (`forces-ui`)

- **The core simulation** — `FieldStore`, the integrator, and the scanner: the particle field on a canvas,
  stepped each frame with the loop paused (and an idle timer armed) whenever the tab is hidden.
- **First-class mass** — `a = F/m`, momentum, and a conserved bound↔free particle reservoir.
- **Two-way density feedback** — bodies bend the field; the field's local density is written back into the
  elements (word glow/grow, weight, motion).
- **Env services** — a scalar-grid field-buffer service (diffuse / propagate primitives), currents
  (carrier waves, bound shimmer, cool→warm render), and formations (eased bias, spread / convergence /
  brownian, the conductor).
- **Zero runtime dependencies**, by policy; `tsc` build, `node:test`, 162 core tests.

### The force catalog (26 forces)

- **The canonical nine designed forces** — `attract`, `emitter`, `spring`, `reflect`, `stream`, `repel`,
  `drag`, `vortex`, `absorb`.
- **Natural primitives** — `gravity` and `charge` over a shared 1/d² kernel, `magnetism` (the Lorentz
  force on a moving charge), and `thermal` (Langevin/Brownian agitation).
- **Extended forces** — `lens`, `gate`, `buoyancy`, `shear`, `crystallize`, `align` (heading steer + boids
  neighbour-mean), `wind` (curl-noise turbulence), `collide` (elastic pairwise), `cohesion` (surface
  tension), plus the integrator-modifier pass with `resonate` and `spotlight`.
- **Presets** — cosmology as composites over the primitives, no new engine code: `blackhole`, `wormhole`,
  `supernova`, `star`, `quasar`, `galaxy`, `nebula`, `tornado`.
- **Orthogonal axes** — conditions (when a force acts), formations (a force applied field-wide), render
  modes (opt-in trails + links), and agents.

### Field agents & reactions

- **The unified target model** — particle, element, and event agents. The element agent brings `data-move`
  transforms + an anchor spring; the event agent lands the event sink.
- **Micro-reaction kit** — a spark draw pass, chromatic shed for `reflect`, and a `recoilImpulse` helper
  (energy made visible).
- **Threads** — glowing connectors between an engaged set.
- **`FieldHandle.burst()`** — a discrete one-shot impulse.

### Surfaces & tokens

- **Colour work** — colour templates (ours / heatmap / infrared / spectrum) and a canonical identity
  colour per force.
- **`cssTokens()`** — design-token emission from the catalog.

### Authoring packages

- **`@forces-ui/elements`** — the `<forces-field>` web-component keystone, plus `<forces-cell>` (an
  in-frame per-force demo surface) and a vanilla `mountField()` adapter.
- **`@forces-ui/react`** — the `<ForcesField>` component + `useForcesField` hook.

### The site (`@forces-ui/site`)

- The self-documenting Astro site: the Field Manual home, `/reference` (the complete definition rendered
  in the UI), and the Lab — paint forces, watch the field react, share the layout. Decorative field
  surfaces marked `aria-hidden`.
