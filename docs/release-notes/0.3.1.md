# 0.3.1

> Shipped 2026-06-12.

`@field-ui/three` joins the published family: the engine runs headless while its swarm and structure
render in a WebGL scene ŌĆö meshes become bodies, and two new read-only seams (`sample`, `readParticles`)

## Added

### `@field-ui/three` ŌĆö bind the engine to a Three.js scene

A new authoring-surface package. It runs the engine headless (`render: 'none'`) and renders the conserved
swarm as a `THREE.Points` layer via `createFieldLayer()` / `FieldLayer` (which implements the full
`FieldHandle`, so `burst` / `flowTo` / `setFormation` / `seed` drive the 3D layer). A `FieldProjection`
seam maps the field to world space ŌĆö `PlaneProjection` (flat heat-relief) or `VolumeProjection` (the real
depth lane, selected by `createFieldLayer({ depth })`). Ships `threeHost()` (the `FieldHost` for a WebGL
scene) and `threeBackend()` (a `RenderBackend` drawing the diagnostic line overlays as scene geometry).
`three` is a peer dependency. (#408)

- **Meshes as bodies.** `layer.addBody(object3d, spec)` registers a `THREE.Object3D` as a field body ŌĆö it
  bends the field and the swarm responds, while `density` / `load` / `lit` feedback flows back onto the
  mesh. The body carries a `data` record (a genome, an inventory), so a mesh can be a meaningful agent,
  not just a force. No core change needed. (#418)
- **Native field visuals.** `vectorField()` (instanced arrow grid) and `streamlineTubes()` (traced flow
  tubes) build scene geometry from `FieldHandle.sample()` ŌĆö the field's structure rendered directly, not
  via particles. (#418)
- **`FieldBodySpec.color` ŌĆö pigment tint on mesh-bodies.** The virtual element carries a body's tint from
  the spec (the `pigment` force's conserved color transport), so a registered mesh can dye passing matter
  with its color. (#418)

### Engine read-outs for external surfaces

- **`FieldHandle.sample(x, y)`** ŌĆö read the live field at a point. Returns the net force a still test
  particle would feel as `{ x, y }` (a thin wrapper over `forceAt`): pure, read-only, samplable at any
  resolution. The seam 3D visualizers consume to build their own field geometry. Mirrored on
  `@field-ui/vanilla` and `<field-root>`. (#418)
- **`FieldHandle.readParticles(out)`** ŌĆö render-agnostic swarm read-out. Copies live particle state into a
  caller-owned `Float32Array` (stride 5: `x, y, z, heat, size`) and returns the count written.
  Zero-allocation and read-only, so a surface with no 2D context can draw the swarm directly.
  `RenderBackend` / `Stroke` are now also exported from `@field-ui/core`. Mirrored on `@field-ui/vanilla`
  and `<field-root>`. (#408)

### Surfaces

- **`overlay` prop for `@field-ui/react`.** `<FieldField>` and `useFieldField` now accept an `overlay`
  prop (`OverlayInput`), lazily creating the fixed, full-viewport, `pointer-events:none`,
  `mix-blend-mode:screen` overlay canvas on first set and removing it on unmount ŌĆö Field Surfaces parity
  with `<field-root overlay="ŌĆ”">`. (#352)
- **`render: 'flow'`** ŌĆö a new underlay mode that draws the dot swarm and the field's streamline arrows
  together in one `<field-root>` canvas, with no separate front surface and no `mix-blend`.
  (`'streamlines'` still draws the arrows alone.) Accepted by `createField`, `setRender('flow')`, and
  `<field-root render="flow">`. (#405)

All additive; the frozen API surface is unchanged (`check:api` green).

## Fixed

- **Homepage framerate ŌĆö free the overlay canvas when idle.** A full-viewport `mix-blend` overlay canvas
  cost every frame the layer below animated, even when empty. It is kept out of the tree unless actively
  drawn. (#405)
- **Smoother scrolling with streamline arrows on.** The `streamlines` / `flow` arrows re-sampled the whole
  force-field grid every frame, though the field only updates on the `measureBodies` cadence (every 6th
  frame). The grid now resamples every 3rd frame and the arrows draw from the cache ŌĆö no flicker, ~3├Ś less
  per-frame work. (#406)
- **The density heatmap is much cheaper per frame.** Its texel grid recomputes on a cadence into a reused
  `ImageData` buffer instead of reallocating every frame, and the full-viewport bilinear-upscale draw is
  suppressed while the page scrolls fast ŌĆö ambient density you read at rest, not detail you track
  mid-scroll. Returns the instant the page settles. (#407)
- **Particles render as a soft glow, not a solid disc.** Each particle is now three concentric additive
  discs ŌĆö a wide faint aura, a mid body, a bright core ŌĆö summing under `lighter` into a smooth radial
  falloff. A sink's captured matter renders again as its dim orbital cloud (an earlier change had removed
  it). (#416)
- **Element consumers ŌĆö inert when hidden.** Consumers are marked `aria-hidden` and their docks / emit
  clones reconcile across rescans instead of duplicating. (#399)
