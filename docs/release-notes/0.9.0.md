# 0.9.0

> Shipped 2026-06-27.

The substrate spreads to a second native runtime. The **Android / Kotlin port reaches full Swift/JS
parity** — engine, platform, both hosts, and the desktop FieldLab — so the field now runs identically
across three planes held to the same cross-plane conformance golden. Alongside it: a **record/replay
seam** that captures a field's frames and reproduces them bit-for-bit (the field as reproducible data), a
run of consumer DX & API additions, and a testing/QA spine (property-based integrator fuzzing, a
frame-driving harness, silent-contract-gap CI). All package API changes are additive — the frozen surface
is intact.

> bump versions or tag.

## Added

### Android / Kotlin port — full parity (`android/`)

- **Kotlin core.** A native Kotlin port of the engine, mirroring the Swift port one-to-one. The
  pure-`kotlin("jvm")` `:fundamental-core` (zero Android deps) ports the core contracts and the **full
  36-force surface** line-for-line. The six deterministic canonical forces are **machine-checked on the
  same conformance golden as Swift** (`syncGolden` pulls `conformance-golden.json` onto the Kotlin test
  classpath; `GoldenConformanceTests` reproduces every `dv` within tolerance). New `android.yml` CI gates
  it.
- **The integrator.** `step()` and its subsystems ported line-for-line from Swift: first-class mass,
  range cull, the modifier contract, conserved attention, carrier-wave currents (linear + circular),
  formations, friction/heat decay, mortal aging, toroidal wrap, real scalar grids, `SpatialHash`,
  `Geometry`/`netField`. Verified by headless `EngineTests` + a deterministic `PerfRegressionTests`
  (1200 particles × 600 frames).
- **Runtime driver + Jetpack Compose host — the engine runs on-device.** `FieldController` (the
  `createField`-equivalent loop) driven by `:fundamental-compose` — `FieldView` (one frame per display
  frame, tap-to-burst) and `Modifier.fieldBody(...)` (a composable becomes a force source, mirroring
  SwiftUI `.fieldBody`) — with a runnable `:sample`. **Verified running on a Pixel 7 / API 35 emulator.**
  compileSdk 34, minSdk 24, AGP 8.7.
- **Public `FieldHandle` API.** `createField(...)` facade over the driver: programmatic `BodyHandle`s
  (`set`/`remove`/`load`/`drain`), `burst`, `flowTo`/`clearFlow`, data atoms, open scalar channels,
  `energy`, `particleCount`, `readParticles(out)` (stride-5 wire format).
- **`:fundamental-platform`** — the six-phase `FrameScheduler` (`discover→read→compute→state→write→
  render`), the six registries, the `FieldPlatform` coordinator, and the `QualityGovernor` / `FieldPerf`
  governors (pure `kotlin("jvm")`, JVM-tested).
- **`:fundamental-android`** — the imperative non-Compose `View`/`Canvas` host (mirror of
  `UIKitFieldHost`): `FieldFieldView` (owns a `FieldController`, drives it from the `Choreographer`) +
  `AndroidFieldHost`. On-device E2E is a follow-up.
- **Compose render modes** — `DOTS` / `TRAILS` / `LINKS` / `GLOW`, all verified on the emulator.
- **Relationship edges** — `addEdge` / `readEdges` port the per-tick edge dynamics, completing the Kotlin
  `FieldHandle` surface.
- **Engine subsystems** — carrier waves + the bound↔free reservoir (§24), reactions & sparks (§23),
  Body-Matter-Interaction (conserved attention, cross-boundary causality, density heatmap), and the
  **locked 64-recipe canon** (decoded at runtime from the shared `data/recipes.json`), plus overlay
  readings.
- **FieldLab for the JVM (`android/lab`)** — a desktop FieldLab over the pure-Kotlin engine drawn with
  Java2D (no emulator): sidebar tour + full 36-force catalog, live canvas, inspector, recipe save/export,
  `path` traces, per-body `data` rings, a `ParticleShape` picker, and a visual-snapshot signature gate.
  Full parity with the Swift lab.

### Substrate — record / replay

- **`recordRun` / `replayRun` / `verifyReplay` / `seededRng` (core, #692).** A pure, headless
  capture-and-reproduce seam: `recordRun(config)` drives a deterministic (seeded-rng + lockstep clock)
  field on a `headlessHost` and captures every frame's particle state via the `readParticles` wire format
  into one compact `Float32Array`; `replayRun` reproduces it bit-for-bit; `verifyReplay` compares;
  `frameAt` slices one frame back out. Foundational slice — an input timeline and on-disk format are
  deferred. JS-only; a port mirror follows.
- **Wire-format contract — `PARTICLE_STRIDE` (5) / `PARTICLE_WIRE_VERSION` (0) (core).** Typed constants
  documenting the `readParticles()` buffer layout so renderers assert the contract.
- **`readParticleChannels(channels, out)` (core).** Column-wise multi-channel particle read into
  caller-owned buffers.
- **`registerOverlay(name, drawFn)` (core).** Register a custom named overlay into the `setOverlay` stack.

### FACM — element influence & agent events (core)

- **Reserved agent-threshold events now dispatched (#686).** The reserved `field:*` events fire as a body
  crosses a metric level — hysteretic + debounced via the shared `Thresholder`, never per-frame:
  `field:saturated`/`field:released`, `field:entered`/`field:exited`,
  `field:attention-shifted`/`field:attention-settled`, and the entropy/memory pairs.
- **Element class-toggle via `data-class` (#687).** The last FACM element-influence cell:
  `data-class="dense:lit, captured:full"` toggles a class while a trigger holds — the declarative no-JS
  counterpart of a `data-on` handler.

### Consumer DX & platform

- **Per-frame coalescing of discrete field events (core, #684).** `FieldHandle.on(...)` occurrences batch
  to at most one delivery per `(source, type)` per frame (last-wins), flushed at frame end.
- **Typed `--field-density` / `--d` via `CSS.registerProperty` (dom, #677).** `browserHost()` registers
  the density channels as typed `<number>` properties, so consumers can transition/animate them on the
  compositor. Feature-detected, idempotent.
- **`textBodies({ observe })` — opt-in ResizeObserver re-annotation (dom, #713).**
- **Single-file (vendorable) build (vanilla, #585).** `dist/standalone.js` (bundled ESM) and
  `dist/standalone.global.js` (IIFE exposing a `Fundamental` global). Size-checked in `check:dist`.
- **`FIELD_VERSION` re-exported from the adapter doors (vanilla / react / elements, #584).** Added to the
  frozen API surface.
- **`useForcesData(records, mapper, options)` React hook (react, #698).** The React wrapper for
  `bindData()` — drives a live field from React state with record → body diffing.
- **Off-main-thread render (C3) — `attachOffthreadRender` (dom).** Transfers a canvas to a Worker and
  drives particle rendering off the main thread. Falls back gracefully without `OffscreenCanvas`.
- **`createFieldPerf({ loaf: true })` — LoAF / long-task lane (dom, #714).** Surfaces `loafCount` and
  `tbtMs` on `snapshot()`. Off by default.
- **`feedback-never-written` lint + `feedbackActivity()` (dom).** The runtime half of the
  silent-contract-gap detector: warns when a `[data-feedback]` body has been bound for many frames yet
  never received a non-zero value.
- **Swift Apple-platform CI** (`swift-apple-platforms.yml`) — iOS Simulator + visionOS build, filling the
  macOS-only gap. **Auto-update-branch CI** — calls `gh pr update-branch` on behind/dirty PRs, unblocking
  the strict merge queue.

## Fixed

- **Android — programmatic bodies build with `feedback=true` (Swift `addBody` parity).** Code-driven
  bodies were constructed without feedback, so their density never rose — silently breaking `addEdge`
  salience, conserved attention, and causality. They now construct with `feedback=true`.

## Performance

- **Property-based fuzz test for the integrator (core, #691).** Random-but-valid field configs stepped
  many ticks assert the invariants across the space: particle **count** conserved, no NaN/Infinity, values
  finite and bounded. Deterministic — every case reproduces from its seed.
- **Frame-driving test harness for element consumers (core test-support, #704).** `frameHarness` drives
  the real `field.ts` frame loop frame-by-frame on deterministic `dt` + seeded `rng`, closing the gap
  where per-frame element wiring never ran under test.
- **Catalog-count doc-drift guard + silent-contract-gap CI (core / CI, #710, #717).** A conformance test
  derives the force count from the catalog and fails CI on any doc stating a different total; an advisory
  `silent-contract-gap` job diffs base vs head for new `[data-feedback]` channels no CSS reads.
