# 0.9.1

> Shipped 2026-06-27.

A same-day follow-up to the 0.9.0 substrate release, focused on consumability: the Swift port becomes a
one-line remote SPM reference, and a full family of runnable consumer example projects (vanilla, Vite,
React web-component, three, and the SSR frameworks — Next.js, Astro, SvelteKit, Nuxt) lands as
proof-of-integration. No published-package API changes — the frozen surface is intact (`check:api` green).

> Draft release notes — paste as the GitHub release body when `v0.9.1` is cut. This file does not itself
> bump versions or tag.

## Added

- **Root `Package.swift` for Swift Package Manager.** Adds a `Package.swift` at the repository root so
  `https://github.com/zachshallbetter/fundamental-engine` can be used as a remote SPM reference without a
  subdirectory path. Exports `FundamentalCore`, `FundamentalPlatform`, `FundamentalVanilla`, and
  `FundamentalSwiftUI` — sources live in `swift/Sources/` and are unchanged.
- **`CITATION.cff` + README citation block (#793).** A machine-readable citation file at the repo root and
  a matching README section, so the project can be cited directly.
- **Consumer example projects — the Track E integration proof (#789–#792).** A runnable `examples/` set
  demonstrating `@fundamental-engine/*` across real toolchains: no-build (`single-file`, `vanilla-cdn`),
  bundled (`vanilla-vite`, `web-component`, `three`), and the SSR wave (`nextjs`, `astro`, `sveltekit`,
  `nuxt`). Each ships its own README and, where applicable, a smoke check.

## Fixed

- **SSR example builds — Next.js, SvelteKit, Nuxt (#792).** Build fixes so the server-rendered example
  apps compile and render cleanly.
- **`three` example smoke check starts the preview server before navigating (#790).** The `smoke.mjs`
  harness now boots the preview server first, fixing a navigation-before-ready race.
