# 0.5.0

> Shipped 2026-06-14.

The **host-agnostic engine** release: the field grows a data plane a non-DOM host can drive — stable
per-particle identity, a discrete event bus, author-owned scalar grids, and gradient-capable density
sampling — alongside engine-stepped agents (the creatures primitive), matter tagging for multiple
ecologies in one field, and the completion of the `field-ui → Fundamental` rebrand across docs.

> Draft release notes — paste as the GitHub release body when `v0.5.0` is cut. This file does not itself
> bump versions or tag.

## Added

- **`FieldHandle.addAgent` — engine-stepped agents (the creatures primitive).** An agent lives in the
  particle pool and is moved by the integrator, so it feels every force the swarm feels — body forces and
  the particle-level ones (`hunt`/`align`/`cohesion`). `maxSpeed` caps it, `species` + `affects` let
  tagged bodies steer it selectively, it edge-bounces (not wraps), and `readParticles` excludes it. The
  engine owns the motion. `@fundamental-engine/three` gains `layer.addAgent(object3d, …)`, the successor
  to `FieldAgent`. Mirrored on vanilla / `<field-root>`. (#438)
- **Matter tagging — multiple ecologies in one field with selective forces.** A body's `data-affects`
  (comma-separated species) restricts its forces to that matter; a `spawn` source's `data-species` stamps
  its tag on emitted matter, so pollen, seeds, and spores share one field, each pulled only by its own
  attractors/sinks. Omit `affects` and the body acts on all matter (bit-for-bit back-compat).
  `@fundamental-engine/three`'s `FieldBodySpec` gains `species`/`affects`. (#444)
- **`FieldHandle.readParticleIds(out)` + `Particle.id` — stable per-particle identity.** Pooled particles
  were anonymous; each now carries a stable monotonic `id`, and `readParticleIds(out)` copies them into a
  caller `Uint32Array` parallel to `readParticles`. The engine carries identity; the host keeps its own
  payload keyed by id. Zero-allocation, read-only. Mirrored on vanilla / elements / three; `Particle.id`
  is optional for back-compat. (#474)
- **`FieldHandle.on(type, cb)` — a host-agnostic discrete event bus.** Plain-data push for discrete
  occurrences (no DOM), returning an unsubscribe. First events: **`absorb`** / **`release`** — a `sink`
  body captured / let go of matter, `{ body, count }`. Detection is lazy: a type with no listener costs
  nothing. Mirrored on vanilla / elements / three. (#473)
- **`FieldHandle.grid(name)` — host-authorable scalar grids.** The engine's field-buffer primitive is now
  public, so an app can lay down and read its own fields the sim composes with (scent map, wear layer,
  goal-attractor). Returns a `ScalarGrid` with `deposit`/`sample`/`gradient` plus new
  `decay(rate)`/`clear()`, created on first access, viewport-sized, advanced once per frame by the mode
  inferred from its name. A force of the same name shares the buffer. Mirrored on vanilla / elements /
  three. (#472)
- **`FieldHandle.sampleScalar(x, y)` + `sampleGradient(x, y)` — smooth, gradient-capable density.**
  `sampleScalar` returns the diffused density scalar ∈ [0,1] (bilinear); `sampleGradient` returns the
  `{x, y}` direction + steepness of increasing density off the same diffused grid (central difference,
  eased-peak normalized), so it stays non-degenerate *at* a source. Both require the heatmap layer, return
  zero when off, and are maintained under `render: 'none'`. Mirrored on vanilla / elements / three.
  (#440, #470)
- **`cssFeedbackSink` — the feedback CSS adapter, named.** The CSS write path
  (`--d`/`--field-density`/`--load`/`--lit`) is now an exported sink, so the DOM door installs it
  explicitly and a non-DOM host cleanly opts out with its own sink. Behavior identical. (#445)
- **Reactive body params — live `strength`/`range`/`angle`/`spin` without a `rescan()`.** A body's hot
  force params are re-read from its element on the measure cadence; only attributes actually present
  override, so preset/intent bodies are untouched. `@fundamental-engine/three`'s `FieldBody` gains
  `set({ strength, range, angle, spin })`. (#442)
- **Adapter type re-exports.** `@fundamental-engine/{vanilla,react}` now re-export `AgentSpec`,
  `AgentHandle`, `AtomPayload`, `FeedbackSink`, and `FeedbackChannels` (and vanilla the `cssFeedbackSink`
  value). (#469)

## Changed

- **Documentation rebrand: `field-ui` → Fundamental, completed.** The code rebrand had renamed the
  published packages to `@fundamental-engine/*` but left the docs on the intermediate `field-ui` name;
  this sweeps ~845 prose/token occurrences across `docs/`, comments, scripts, and the Swift
  port, and drops the `field-ui-` prefix from the 13 canonical docs. The `forces-ui` lineage and the
  CHANGELOG/MIGRATION history are preserved as historical record. No runtime change. (#455, #466)

## Fixed

- **`@fundamental-engine/three` samplers — EMA-smoothed vector field + core field-line tracing.**
  `vectorField` normalized arrows by the raw per-frame peak (jittering as the peak shifted); it now eases
  the peak (#422). `traceStreamline` re-walked forward-only; it now delegates to core's `traceFieldLine` —
  bidirectional, loop-closing, with a turn budget (#421).
- **Frame-rate-independent particle motion.** `env.dt` was a flat `1`, so the sim ran 2–4× faster once the
  perf work lifted the homepage to 60–120fps. `dt` is now the real frame interval normalized to a 60fps
  baseline (clamped so a stall can't teleport matter); position is dt-scaled, forces/friction stay
  per-frame by design. Mirrored to the Swift port. (#434)
- **Particle glow is now crisp points.** The three-disc soft glow and the older heat-scaled halo bloomed
  into large overlapping rings wherever an accretion sink heats a cluster. Particles now draw as a crisp
  core with a single fixed ~1px bloom. (#434)
- **The density heatmap no longer reacts to scroll.** The scroll coupling is removed entirely — a
  continuous ambient layer served by the existing compute throttle (recompute every 3rd frame). (#460)
- **Engagement listeners no longer accumulate on a long-lived field.** Each rescan now drops disconnected
  `[data-hot]` engagements. (#452)
- **React adapter forwards the full `FieldOptions`.** `<FieldField>`/`useFieldField` had dropped `depth`,
  `heatmap`, `overlayBackend`, `rng`, `now`, and `feedbackSink`; all `FieldOptions` are now forwarded.
  (#468)
- **Lifecycle teardown closes registry + observer leaks.** `platform` destroy prunes stale registry
  entries and disconnects observers; three's layer tears down its body registry and reuses overlay GPU
  buffers. (#463)
- **Rebrand stragglers in user-facing engine strings.** The `inspect` recipe's `intent`, the
  system-report heading, and the canvas-context error/warn messages still said "field-ui"; renamed to
  "Fundamental". (#458). Stale `@forces-ui` alias claims removed from docs (#467).

## Performance

- **Reuse draw/flow scratch instead of allocating per particle per frame.** The core draw and flow paths
  allocated scratch (`flowBias`/`particleRGB`) per particle per frame; the hot loops now pass shared
  module scratch via internal write-into variants — the public wrappers and the math are bit-for-bit
  unchanged. (#463)
