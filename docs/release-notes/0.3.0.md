# 0.3.0

> Shipped 2026-06-12.

The field goes multi-plane and multi-dimensional: a native **Swift port** mirrors the whole npm family
(byte-equivalent at z = 0), an opt-in **z lane** gives the engine a real third axis (flat stays
bit-for-bit exact), recipes stop describing and start **executing**, and the **physics workover** begins.
Two structural seams land — `RenderBackend` and injectable `rng`/`now` — as the foundation for the WebGPU
frontier and deterministic record/replay.

> Draft release notes — paste as the GitHub release body when `v0.3.0` is cut. This file does not itself
> bump versions or tag.

## Added

- **Swift port of the engine.** `FieldUICore` / `FieldUIPlatform` / `FieldUIVanilla` / `FieldUISwiftUI`
  mirror the npm family 1:1, plus the **FieldLab** macOS showcase app — byte-equivalent to the JS engine
  at z = 0. Lives in `swift/`, versioned separately from the npm packages.
- **The optional z lane — a not-required third axis** (`@field-ui/core`). `createField({ depth: 300 })`
  seeds matter through a shallow volume behind the page; z integrates/damps/wraps toroidally, the `c` cap
  bounds full 3D speed, dots recede with depth, and the neighbour/radial forces go truly 3D. **Flat is
  exact:** with no `depth` every z term multiplies away and the engine matches prior behavior bit-for-bit
  (`z-axis.test.ts`). No public signatures change. See `docs/engine-reference/z-axis.md`.
- **`background: 'transparent'`** — additive `FieldOptions.background` (`'opaque'` default) clears the
  underlay to transparent so bright matter composites over 3D scenes, images, or light pages; trails fade
  to transparent (`destination-out`). Live via `setBackground`, `<field-root background>`, and the React
  prop.
- **Recipes execute their declarations.** `compileRecipe` derives an executable render plan (underlay
  matter mode + additive overlay stack + heatmap toggle; unmappable layers NAMED in `plan.unapplied`);
  `applyRecipe` gains a structural `field` target it drives and releases on `destroy()`. `BodyRecipe.when`
  is the new executable gate, compiled to `data-when` and validated against registered condition ids.
- **RenderBackend — the drawing seam (#373).** Structural contract between engine and drawing surface
  (`size`/`clear`/`segments`/`polyline`/`rect`/`text`) with a Canvas 2D default; the overlay (all eight
  readings) now renders exclusively through it, and `createField({ overlayBackend })` accepts any
  conforming impl — the seam WebGL/WebGPU builds on.
- **Injectable rng + wall clock (#371).** Every random draw flows through one source —
  `createField({ rng })` (default `Math.random`), carried as `env.rng` — making runs reproducible
  (record/replay); `createField({ now })` (default `performance.now`) completes the three-clocks
  separation (wall/frame/sim).
- **Traced field lines + the `gravity-field` preset.** The `field-lines` overlay draws real field
  *structure as curves*: new `fieldLineSeeds`/`dipoleSeeds`/`monopoleSeeds` (core) seed each field-bearing
  body by geometry and `traceFieldLines` follows the net field. `FIELD_BEARING_TOKENS` (`magnetism`,
  `charge`, `gravity`) is canonical. The experimental **`gravity-field`** recipe joins
  `EXPERIMENTAL_RECIPES`.
- **`FieldLineOpts.maxTurns`** — a turning budget for the tracer, counting cumulative heading change in
  revolutions; caps degenerate orbits (~81× faster in the Swift CoreGraphics renderer). `Infinity`
  (default) preserves unbounded behavior exactly.
- **Additive overlay readings.** `setOverlay` (core, `<field-root overlay>`, vanilla) accepts a stack —
  array or space-separated attribute — drawn in order. Five new line/text diagnostics: `grid` (lattice
  deformation), `temperature` and `energy` (iso-contours), `path` (streamline curves), `data` (numeric
  `--d` readouts).
- **Contour typography primitive** (`@field-ui/platform`). `contourPathData(font, text, size)` lays out
  glyph-outline SVG path data; `contourSvgFor(el, font)` binds an aria-hidden contour-ring SVG driven by
  the body's live `--d`/`--load`. Caller supplies the parsed font (`ContourFont` contract) — field-ui
  stays zero-dependency.
- **`<field-root heatmap>`** — the density heatmap is now a declarative, observed attribute (toggles via
  `setHeatmap`), alongside `mass`/`attention`/`causality`.
- **The physics workover begins** — a designed/natural/hybrid substrate for physical coherence without
  losing the designed feel; plan + as-built audit in `docs/engine-reference/physics-workover.md`. Ships
  across v0.3–v0.6.

## Changed

- **Supernova ejecta are now persistent, escaping field matter** (`@field-ui/core`, `accretion.ts`).
  `releaseCaptured` clears each released particle's `age` and ejects it just past `absorbR` along its
  bearing, so the sink no longer re-captures its own ejecta. A `spawn → sink → supernova` loop now
  visibly conserves and repeats at a real period (≈9 frames vs ≈1). No-op for the ageless base pool.

## Fixed

- **One source of truth for reduced-motion and page-visibility** (`@field-ui/platform`). Four scattered
  `matchMedia` calls and two `document.hidden` reads consolidate into `env.ts`
  (`prefersReducedMotion()`/`pageHidden()`), SSR-safe, with `setEnvOverrides`/`clearEnvOverrides` as a
  clean test seam.
- **Platform registries close their exits.** `FeedbackRegistry`, `RelationshipRegistry`, and
  `StateRegistry` leaked entries for departed elements; each now prunes disconnected elements and gains
  explicit `unregister(element)`.
- **Warp pair ghost (#368a)** — a removed `data-pair` target now clears `pairBody`/`warpHas` so the
  wormhole closes instead of relocating matter to a detached node.
- **Docked element removal (#368b)** — `updateMovers` detects `!el.isConnected` and clears `mv.docked`, so
  a removed `[data-dock]` mover no longer leaves the sink believing it holds the element.
- **Heatmap buffer persists across disable/enable (#369)** — `setHeatmap(false)` now `heatmap.clear()`s
  before releasing the buffer so no stale density bleeds into the next session.
- **Streamlines arrow-field pulsing eliminated.** The underlay `streamlines` render and all three overlay
  arrow modes normalized to the raw per-frame peak, flashing on any `maxMag` shift; both now track an
  independent EMA (rise 0.3 / decay 0.1).
- **`priority-well` recipe note corrected** — `density` writes as `--d` on `data-feedback` bodies, not the
  host-supplied `--field-density` lane.
- **The home manual's last two untraced stages now trace real runs** — `fieldflow` pairs with a live
  magnet and `warp` wires its pair target headlessly; both gained demo-accuracy tests.
