# F1.9 — Stage-1 findings (scientific synthesis)

> **Status: Stage-1 synthesis, 2026-07-19.** This is the scientific record of Stage 1, not a changelog.
> Claims use the repository's evidential language — *demonstrated in the current model*, *established for
> the tested substrate*, *supported by the ablation*, *conditionally derived*, *unresolved beyond the
> tested domain*. No claim here is called a theorem. Historical hypotheses and falsified positions are
> preserved as stated, not rewritten.

## 1. Initial hypothesis

Stage 1 opened with the working kernel

```
K = ⟨Entities, State, Relations, Operations, Dynamics, Projection, Invariants⟩
```

and a concrete first test (original F1.1): **a typed kernel core hosting an existing `FieldPattern`,
proven by three equivalences** (operational, structural, invariant). The premise was that
`CompiledPattern` was, or could be treated as, a world declaration.

## 2. F1.1 falsification (preserved)

The integration audit **falsified that premise**. In the current engine:

- `CompiledPattern` is *authoring/runtime configuration* — a stringly `data-*` attribute bag, render and
  feedback plans — not a declaration of a world;
- live state is scattered across ~20 closure variables inside `createField`, not a central object;
- snapshots are lossy (no velocities, particle ids, registry, grids);
- lawful evolution exists **only as executable `apply(b, p, env)` force code** in a per-field registry;
- therefore hosting the running pattern would have required retaining an opaque field instance and
  delegating `tick()` — a delegation harness, not hosting.

The original F1.1 acceptance criterion is **superseded, not deleted**. Its falsification is the Stage-1
result that reshaped everything after it: *the kernel sits above the runtime, not around it.*

## 3. World and DynamicsContract separation

Stage 1 was reframed (A′) as:

```
World  →  DynamicsContract  →  ExecutionSubstrate      (FieldRuntime is the first substrate)
```

- **F1.2** built a generic, field-free `World` declaration (identity, envelope, entities, relations,
  invariants, projections). A no-escape-hatch guard enforces that the kernel imports nothing
  field-specific and holds no substrate instance.
- **F1.3** built the `DynamicsContract` execution boundary: `identity · executionKind · capabilities ·
  determinism declaration · initialize / advance / snapshot? / restore?` returning typed
  `DynamicsResult`, with a typed evidence channel whose `unresolvedInterpretations` **record** rather
  than assert interpretation. There is deliberately no `declarative: boolean`.

The same kernel executes an `opaque-native` field substrate and an `interpreted` non-field substrate
without modification — *demonstrated in the current model*.

## 4. Conditional substrate equivalence (F1.4)

With the raw field path as the authority (constructed independently of the adapter) and both paths built
from one fixture-owned `FieldConstruction`, the field behind the contract was compared **at every
transition**:

- **Result:** observable execution is preserved — *established for the tested substrate*.
- **Conditions (recorded, not assumed):** injected RNG, injected clock, controlled `dt`, body ordering,
  environment, host geometry. The pass is explicitly **conditional**, never unconditional.
- **Honesty artifacts:** snapshot fidelity is `partial-observable` (non-restorable); structural coverage
  classifies velocity/accumulated force as `lossy` and lifecycle callbacks as `unavailable`; nothing was
  upgraded to `represented` to make a test pass.
- Negative fixtures demonstrated sensitivity, including a case where a final-state-only check
  **false-passes** while per-transition comparison catches the divergence.

## 5. Declarative versus opaque dynamics boundary (F1.5)

A minimal typed expression IR (closed op union; no callbacks, `eval`, `Function`, or script) was tested
against a representative force corpus.

**Outcome: `partial-with-opaque-extensions`.**

```
Dynamics  =  Declarative Region  +  Opaque Execution Region
```

- **Declarative:** distance-falloff, relation-scaled, environment-constant, range-threshold, composition,
  parameterized-constant, linear-time; plus accretion with **explicitly surfaced** prior state.
- **Opaque:** nonlinear-time (`unsupported-input`), closure-captured state (`hidden-mutable-state`),
  host/callback dependence (`callback-dependence`).
- The IR was **not** expanded to force success; the stop conditions were respected.

*Supported by the experiment:* non-trivial laws are representable as data, and the force system as a
whole is not.

## 6. Ω_sys derivation (F1.6)

`evaluateOpportunity(context, operation)` computes system-relative opportunity from lower structures
only: `Ω_sys = f(X, 𝒪, Cap, Auth, Reach, Π, H)`. It returns per-predicate results
(`domainValid · capable · permitted · enabled · reachable · exposed · signaled · reversible` +
`recoveryPaths` + `failedPredicates` + evidence retaining the version envelope). It contains **no**
belief, perceived availability, interpretation, expectation, confidence, experience, or strategy — that
exclusion is asserted by test. `Capable` (can) and `Permitted` (may) are separate, and every permission
requires an `authoritySource`.

## 7. Episode derivation (F1.7)

`detectEpisodes(trace, ⟨B, T, C, R, I⟩, worldParticipants)` returns **conditional** findings — *given
these parameters, this trace satisfies the candidate-episode rule* — with candidate episodes, episode
participants, supporting pairs, boundary used, coupling/recurrence basis, determinacy, **alternate
segmentations**, failure reasons, and evidence. Coupling is separated from shared cause **structurally**
(directed `from→to`), and the world-participant / episode-participant distinction is preserved.

## 8. Ablation method (F1.8)

Four ratified forms — **removal · collapse · substitution · reconstruction** — were run by *executing*
the real F1.5–F1.7 derivations with ablated inputs, not by asserting stored verdicts. Every record
carries hypothesis, transformation, fixture, expected distinguishing case, observed result, failed
capability or preserved behavior, evidence, classification, and K/K₀ implication. A negative fixture
asserts the harness does **not** report loss where none occurs (invoked operations and observed edges
reconstruct cleanly).

## 9. K versus K₀ result

```
K   = ⟨Entities, State, Relations, Operations, Dynamics, Projection, Invariants⟩
K₀  = ⟨X, Θ, Π, V⟩          X carries typed entity / relation / operation structure
```

**K₀ is supported by the ablation — as a *representational* reduction, not a semantic one.** Entities,
Relations, and Operations fold into `X` as typed structures; they are **not derived away**. Every
distinction they carry survived ablation and must still be represented somewhere.

All seven K elements were retained (as elements or typed structures inside `X`). Four constructs were
established as **not** kernel elements: Opportunity, Interaction episode, Capability/Authority, and the
DynamicsContract.

## 10. Accepted reductions

| Reduction | Basis |
|---|---|
| **Opportunity → derived** (`derived-complete`) | Fully reconstructed from state, operations, capability, authority, projection, reachability, history. Removed from the kernel. |
| **Interaction episode → derived** (`derived-conditional`) | Reproduced from transitions + world participants under a declared contract; conditional on ⟨B,T,C,R,I⟩. Removed from the kernel. |
| **Capability / Authority → typed constraints** | Consumed by derivations; neither is a kernel element (consistent with ratified M1.5-04). |
| **DynamicsContract → execution boundary** | A runtime interface enabling substrate interchange; **not** promoted to an eighth primitive. |
| **Entities / Relations / Operations → typed structures in `X`** | A representational fold (7 → 4), with all distinctions preserved. |

## 11. Rejected reductions

| Attempted reduction | Result | Distinguishing evidence |
|---|---|---|
| Dynamics → declarative State | **rejected** (`non-substitutable`) | 3 of 11 corpus laws have no declarative representation (hidden state, host/callback, nonlinear time). |
| Relations → transitions | **rejected** (`collapsible-with-loss`) | A declared latent relation never appears in any transition; relation *type* is not recoverable. |
| Operations → observed transitions | **rejected** (`non-substitutable`) | A latent, never-invoked operation is absent from any transition-derived vocabulary. |
| Projection → observation | **rejected** (`non-substitutable`) | Identical state under two projections yields different `Ω_sys` availability. |
| Invariants → guards inside Dynamics | **rejected for opaque substrates** | The field substrate declares `inspectInternalState=false`, `declareTransitionLaw=false`; no guard-installation point exists. |
| Capability ⊕ Authority → one allowance flag | **rejected** (`collapsible-with-loss`) | capable-not-permitted and permitted-not-capable become indistinguishable; the diagnosis is lost. |

## 12. Necessary distinctions

Each is retained because a minimal fixture distinguishes it:

- **can vs may** — `Capable` / `Permitted`.
- **exposure vs signaling** — invocability vs discoverability.
- **reversibility vs recovery** — an irreversible operation may still have a recovery path.
- **world participant vs episode participant** — a latent authority holder is never an episode participant.
- **coupling vs shared cause** — structural, survives the most permissive thresholds.
- **declarative vs opaque dynamics** — the F1.5 boundary.
- **observed transition vs available / latent operation** — invoked ≠ invocable.
- **latent relation vs observed edge** — persistence is not reconstructible from activity.

## 13. Limits of the evidence

- The F1.5 corpus is **representative laws modeled on the engine's force categories**, not the exact 36
  shipped force functions. Category representability is characterized; per-force coverage is not.
- F1.4 equivalence is **conditional** on declared controls and covers only the **observable** projection
  (velocity and accumulated force are `lossy` and were not compared).
- Only **one real substrate** (the field) plus one synthetic substrate were characterized under the
  contract. Contract generality beyond these is *unresolved beyond the tested domain*.
- JS reference plane only; native ports are deferred (R4).
- Boundary justification for episodes remains **analytic/empirical**, not runtime-decidable — the runtime
  reports conditional findings and alternates, and nothing more.
- The K₀ fold is representational and **untested at scale**.
- No empirical or human-subject validation is claimed anywhere in Stage 1.

## 14. Implications for Stage-2

1. **Stage 2 (projections as executable contracts) is well-founded.** Projection was shown
   non-substitutable and materially outcome-changing, so a `ProjectionContract` with typed claims is
   justified rather than speculative.
2. **The three-class property language matters.** Because projection changes derivation results,
   mechanically-decidable and model-checkable claims must be separated from empirically-testable ones,
   which may only produce an *unresolved empirical obligation* — never a pass.
3. **Substrate plurality is the next real test of the contract.** A second non-trivial substrate should
   be characterized to test whether `DynamicsContract` generalizes or merely fits the field.
4. **Invariants stay kernel-side** in the F2 validator design; guard-injection is not available for
   opaque substrates.
5. **Schema work follows the fold:** if `X` carries typed entity/relation/operation structure, the world
   schema must express latent relations, relation type, and latent operations — precisely the things
   ablation showed are not reconstructible from activity.
6. **Do not begin Stage 2 implementation** on the strength of this synthesis alone; F2.1's property
   language is the gated entry point.
