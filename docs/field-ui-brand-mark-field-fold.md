# field-ui Brand Mark Concept: Field Fold

## Core idea

The mark shows an interface body bending a shared field, while the field returns state to the body.

It is based on the system principle:

```txt
Elements bend the field.
The field bends them back.
```

## Visual components

| Component | Meaning |
|---|---|
| Central vertical body | A DOM body, component, glyph, card, or interface element |
| Curved field lines | `field()` structure, not particle paths |
| Accent core | reciprocal feedback: density, attention, heat, or charge returning to the body |
| Four small particles/agents | particles, elements, users, data, or relationships participating in the field |
| Subtle outer contours | shared context layer surrounding the interface |
| Accent ticks | cause vectors: `apply()` and feedback are active, not decorative |

## Design rules

- The mark must remain legible in one color.
- The accent core may be cyan, but the form must not depend on color.
- The mark should work as SVG, favicon, app icon, docs icon, and loading mark.
- The shape should imply field structure without becoming a particle wallpaper.
- The central body should keep the interface at the center of the identity.
- The mark should not look like a generic atom, magnet, AI sparkle, or orbit logo.

## Suggested name

**Field Fold**

The name works because the mark shows the interface folding the field and the field folding back into the interface.
