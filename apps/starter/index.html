The first five minutes

A plain Vite + HTML app that uses Fundamental the way any outside project would: one declarative <field-root>, then the same recipe applied three ways. No framework, no canvas code, no per-element wiring. Toggle reduced motion at any time — the page stays meaningful with the motion off.

1 · Declarative

Put a real body token on any element with data-body. These are the priority-well recipe's own tokens — attract and gravity — so the elements participate the moment the page loads.

Anchor

2 · applyRecipe()

Apply priority-well to elements already on the page. The recipe frames which metrics it tracks; it writes them back as --field-* custom properties the CSS reads — here --field-priority sets each task's weight.

3 · bindData()

Hand bindData() your data and a mapper. Each record becomes a body, its mapped metrics become field state, and the same recipe supplies the behavior. Add and remove records — the diff is deterministic and removed records decay before they leave.
